# Document Patch Adapter — LF Profile Creator

## Purpose

Translate approved profile pack candidates into human documentation updates only after review gates are passed.

## Document layer

Google Docs remains the human-readable documentation layer. GitHub remains the technical pack layer.

## Rules

- Do not patch documents before approval.
- Do not create duplicate profile documentation.
- Use controlled patch flow with clear anchors and verification.
- Keep technical source in GitHub and human documentation in Google Docs aligned after approval.

## Blockers

- Missing approval.
- Missing source authority.
- Duplicate active profile.
- Runtime or production general enablement request.
