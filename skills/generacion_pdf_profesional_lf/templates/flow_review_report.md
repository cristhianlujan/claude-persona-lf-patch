# Flow Review Report Template

Use for onboarding, journey or operational flow review.

Required pages:
1. Cover.
2. Flow metadata and scope.
3. Phase map.
4. Screen matrix.
5. Critical risks.
6. Legal and integration dependencies.
7. Decision board.
8. Open blockers.
9. QA evidence.

Flow rules:
- Do not compress more than seven nodes in one page.
- Split long flows into phases.
- Use one card per screen.
- Each card shows screen name, user action, system action, risk and status.
- Archive screens go to appendix.

Fail if the flow is unreadable, too dense, or lacks risk/control separation.
