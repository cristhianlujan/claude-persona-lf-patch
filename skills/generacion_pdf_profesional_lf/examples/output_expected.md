# Expected output example — PDF profesional LF

## Entregables esperados

1. `brandbook_lf.pdf`
2. Fuente editable segun pipeline:
   - `brandbook_lf.docx`, o
   - `brandbook_lf.pptx`, o
   - `brandbook_lf.html` + assets.
3. Checklist QA.
4. Readback operativo.

## Estructura esperada del PDF

1. Portada.
2. Resumen ejecutivo.
3. Contexto y objetivo.
4. Paleta de colores.
5. Tipografia.
6. Grilla y espaciado.
7. Componentes base.
8. Componentes extendidos.
9. Flujos de uso.
10. Reglas visuales.
11. No-usos.
12. Checklist de implementacion.
13. Evidencia y pendientes.

## Readback esperado

```text
Estado: CANDIDATO / REVISABLE
Runtime: NO_HABILITADO
Impacto automatico: BLOQUEADO
PDF final: generado
Fuente editable: entregada o NA justificado
Paginas revisadas visualmente: portada, indice, paginas con tablas, cierre
QA visual: PASS / PASS_CON_OBSERVACIONES / RETURN_TO_WORKER / BLOCKED
Pendientes: listados, no ocultos
```

## No aceptado

- PDF solo texto.
- PDF sin portada profesional.
- PDF sin componentes visuales.
- PDF sin fuente editable cuando sea necesaria.
- PDF sin QA visual.
- PDF cerrado como aprobado sin revision del owner.
- PDF con texto cortado, solapes, tablas ilegibles o bajo contraste sin corregir.
