# Expected output example

## Entregables

1. `brandbook_lf.pdf`
2. Fuente editable: `brandbook_lf.docx` o `brandbook_lf.pptx` segun pipeline seleccionado
3. Checklist QA
4. Readback operativo

## Estructura esperada del PDF

1. Portada
2. Resumen ejecutivo
3. Paleta de colores
4. Tipografia
5. Componentes base
6. Componentes extendidos
7. Flujos de uso
8. Reglas visuales
9. No-usos
10. Checklist de implementacion
11. Evidencia y pendientes

## Cierre esperado

```text
Estado: CANDIDATO / REVISABLE
QA visual: PASS con observaciones menores o PASS completo
Fuente editable: entregada
PDF final: entregado
Pendientes: listados, no ocultos
```

## No aceptado

- PDF solo texto.
- PDF sin portada profesional.
- PDF sin componentes visuales.
- PDF sin fuente editable cuando sea necesario.
- PDF sin QA visual.
- PDF cerrado como aprobado sin revision del owner.
