# Example input brief — PDF profesional LF

## Solicitud

Crear un PDF ejecutivo para revisar un Design System LF y presentarlo al owner.

## Objetivo

Convertir insumos visuales y decisiones de marca en un brandbook/reporte PDF profesional, claro y verificable.

## Audiencia

- Owner.
- Direccion.
- Equipo operativo.

## Tipo de PDF

Brandbook + sistema de componentes + guia de uso.

## Estilo visual esperado

Fintech corporativo, premium, sobrio, visual, no basico.

## Fuentes disponibles

- Imagenes de referencia provistas por el usuario.
- Paleta derivada visualmente.
- Decisiones simples tomadas por el asistente.
- Activos LF consultados por Router si aplica.

## Restricciones

- No inventar datos no visibles.
- Registrar supuestos.
- Conservar fuente editable cuando corresponda.
- Entregar PDF revisable.
- Incluir paleta, tipografia, componentes, usos y no-usos.
- No declarar validado ni vigente.

## Pipeline esperado

```text
Brief -> clasificacion brandbook -> HTML/CSS o PPTX 16:9 -> render PDF -> QA visual -> correccion -> readback
```

## QA esperado

- Portada profesional.
- Indice o estructura clara.
- Componentes visuales.
- Tablas legibles.
- Cards, semaforos o flujos cuando aplique.
- Footer con version/estado/pagina.
- Readback de paginas, fuente editable y pendientes.
