# Priority Checklist

## P0 — Governance safety
- No official write without approval.
- No runtime enablement.
- No production general enablement.

## P1 — Completeness
- Pack contains all minimum files.
- Output validates against schema.

## P2 — Anti-basic-output
- Prose-only output is blocked.
- Self-repair gives a complete pack path.

## P3 — Handoff
- Quality Pack can review without inventing missing structure.
