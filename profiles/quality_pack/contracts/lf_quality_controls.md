# Contract — LF Quality Controls

Status: CANDIDATE_READ_ONLY / SANDBOX
Applies to: Quality Pack reviews for LF and MarketPlace Libertad Financiera artifacts.

## LF safety principles
- Clarity before conversion.
- Guidance/accompaniment before selling.
- User autonomy before pressure.
- No debt shame, fear or collection tone.
- No false urgency, countdowns or scarcity.
- No guaranteed debt elimination claims.
- No visual alarm patterns for debt.

## Debt-context checks
Quality Pack must explicitly inspect:
- Does the artifact feel like support rather than collections?
- Does it avoid red/danger cues for debt?
- Does it avoid pushing products before diagnosis or clarity?
- Does it reduce anxiety through warm_surface, spacing and clear next action?
- Does it avoid too many choices in one screen?
- Does it preserve the user's agency?

## Visual/product checks
For UI, prompt or render specs, Quality Pack must verify:
- It does not become landing page when the task is product screen.
- It does not become dashboard when the task is route/onboarding.
- It does not become catalog before recommendation stage.
- It does not include internal workers, scores, PASS, GitHub, Supabase, logs or sandbox traces.
- It does not add unrequested CTA, footer, testimonial, metric or login.

## Hard fail
Any artifact that uses pressure, shame, fake urgency, guaranteed outcomes, debt alarm visuals or unapproved internal metadata must be blocked.
