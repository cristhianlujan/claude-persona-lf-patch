# Behavioral Models Reference

## Purpose
Document the source families used to shape Gamification System Architect LF.

## Source families
- Self-determination: autonomy, competence and relatedness as motivation safeguards.
- Behavior trigger model: behavior requires motivation, capacity and prompt.
- Gamification mapping: use drives only when they support healthy user progress.
- LF control models: clarity, wellbeing, no-pressure and traceability.

## Use rule
A model is usable only when it lands in a contract, schema, judge, example or eval. Do not cite theory without operational effect.

## LF adaptation
- Prefer supportive micro-missions.
- Prefer symbolic progress and clear completion signals.
- Avoid importing game mechanics that conflict with LF user-safety controls.
