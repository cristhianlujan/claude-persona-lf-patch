# Screen Mockup Contract

## Regla

Si el entregable contiene pantallas, cada pantalla clave debe representarse como mockup de app, no como texto plano.

## Mockup minimo

- frame mobile o desktop segun contexto;
- header con marca del proyecto;
- estado o progreso si aplica;
- campos, cards, botones y mensajes segun screen_visual_specs;
- colores del proyecto;
- errores y estados relevantes;
- nota de estado CANDIDATO si el flujo no esta aprobado.

## Overall source

Para Overall leer:

- overall_design.v_onboarding_comments_summary
- overall_design.screen_components

## Bloqueos

- mockup sin frame;
- usar colores genericos;
- ignorar brand_policy;
- ignorar blocked_visual_patterns;
- mostrar progreso donde screen_visual_specs dice no mostrar;
- presentar una tabla en lugar de mockup cuando la pantalla es clave.
